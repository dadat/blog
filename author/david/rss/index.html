]]>http://localhost:2368/running-ghost-0-7-2-on-windows/d415e2b6-ac53-4f9e-9c1d-a29d6f2e1c33Mon, 07 Dec 2015 05:42:45 GMTIt took me a few hours of tinkering with ghost-0.7.2, node.js, python, and finally buster, but deploying static sites created with ghost on Windows is finally easier.

Since the only way to generate and deploy Ghost's static pages is with buster, doing it in Windows 10 is a pain in the ass; buster was not written with Windows users in mind.

Downloading the installers

First you will need the following:

  • Git Bash
  • node.js
  • Python with version below 3.0
  • pip - installed with either chocolatey or pip 7.1.2 here

The Process

Other people will have various ways of setting up their installation, so be sure to have at least a familiar understanding of command line and command line tools.

Step 1:

For Git Pages projects:

Create a new repository called blog, add a branch called "gh-pages". This will serve as your project page for Git Pages. It will look like http://username.github.io/blog/

Open up git bash and clone your repository:

$ git clone https://github.com/username/blog.git

Check to see if the installations were successful by typing in your command line:

python --version

node --version

npm --version

If they output their respective versions, then the installation is complete.

Download Ghost and extract it onto your desktop. Enter the newly created folder and open git bash.

$ npm install

Let ghost install its packages, this will be in a folder called node_modules.

$ npm start

A local instance of your site will now be running on http://localhost:2368.
Now you can play with the settings on http://localhost:2368/ghost

Modifying Buster for Windows

Buster is a tool for generating and deploying static pages. It can only be used while you are running a local instance of your site.

Get buster here: https://github.com/axitkhurana/buster

IMPORTANT:

This build of buster is not compatible with Windows, here is a buster.py file which will fix routing and file name issues you might encounter.

Link to file

Copy or download this and replace buster.py inside /buster/buster.

Optional:

If you wish to route your local git repository with buster, change directory into /buster/buster and open up git bash. Type in:

$ python buster.py setup

Enter your repository's SSH link. It will look like this: git@github.com:username/blog.git

Buster will then create a folder called static.

If you do not with to do this for reasons like missing dependencies or broken links, then follow the next step.

Make sure you have your local site up and running. Enter /buster/buster, open git bash and type in:

$ buster generate --domain=http://127.0.0.1:2368

This will generate your local site into html pages inside a new folder called "static".

If you have followed the optional step, then you may deploy your site directly into your github repository. Input this on git bash while outside of your static folder:

$ buster deploy --dir=/yourfoldernamehere/

If you did not follow the optional steps, just copy and paste the contents of static into your repository folder and push onto GitHub.

Check which files have been added, open up git bash inside your repo folder.

$ git status
$ git add -A
$ git commit -m "Commit message here"
$ git push origin gh-pages

This will push all of your files into the gh-pages branch of your repository. Wait a few seconds for GitHub's servers to refresh and visit your Git Pages hosted site on: http://username.github.io/repositoryname

Enjoy your new site. If you have a domain, just look up CNAME and domains on google.

Getting Started

Ghost uses something called Markdown for writing. Essentially, it's a shorthand way to manage your post formatting as you write!

Writing in Markdown is really easy. In the left hand panel of Ghost, you simply write as you normally would. Where appropriate, you can use shortcuts to style your content. For example, a list:

  • Item number one
  • Item number two
    • A nested item
  • A final item

or with numbers!

  1. Remember to buy some milk
  2. Drink the milk
  3. Tweet that I remembered to buy the milk, and drank it

Links

Want to link to a source? No problem. If you paste in a URL, like http://ghost.org - it'll automatically be linked up. But if you want to customise your anchor text, you can do that too! Here's a link to the Ghost website. Neat.

Quoting

Sometimes a link isn't enough, you want to quote someone on what they've said. Perhaps you've started using a new blogging platform and feel the sudden urge to share their slogan? A quote might be just the way to do it!

Ghost - Just a blogging platform

Working with Code

Got a streak of geek? We've got you covered there, too. You can write inline <code> blocks really easily with back ticks. Want to show off something more comprehensive? 4 spaces of indentation gets you there.

.awesome-thing {
    display: block;
    width: 100%;
}

Ready for a Break?

Throw 3 or more dashes down on any new line and you've got yourself a fancy new divider. Aw yeah.

Advanced Usage

There's one fantastic secret about Markdown. If you want, you can write plain old HTML and it'll still work! Very flexible.

That should be enough to get you started. Have fun - and let us know what you think :)
